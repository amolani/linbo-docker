  <name>Task 1: Rewire main flow with checkpoint guards, pre-flight, and structured errors</name>
  <files>containers/init/entrypoint.sh</files>
  <action>
Restructure the main flow section of entrypoint.sh (everything after the function definitions) to use the helper functions from Plan 01. The existing helper/utility functions (fetch_packages_index, parse_package_info, provision_boot_files, provision_gui, provision_kernels, provision_themes, etc.) stay as-is, but calls to them in the main flow are wrapped with checkpoint logic and error handling.

**Main flow restructure (replaces current lines ~510-634):**

**Step 0: Setup**
- Print startup banner (existing: "=== LINBO Init (APT-based) ===")
- Show target/kernel/APT dirs (existing)

**Step 1: Pre-flight checks**
- `check_write_permission "${LINBO_DIR}"` -- fail early if volume not writable
- `check_disk_space` -- fail early if <500MB
- `check_dns` -- fail early if DNS broken
- Wrap each in `if ! check_X; then exit 1; fi` pattern (set -e immune)

**Step 2: FORCE_UPDATE handling**
- If FORCE_UPDATE=true: call checkpoint_clear_all, print "Forcing full update -- all checkpoints cleared"

**Step 3: APT index fetch (checkpointed as "apt-index")**
- If checkpoint_exists "apt-index": print skip message
- Else: call fetch_packages_index with structured error on failure, set checkpoint
- Parse package info for both LINBO_PKG and GUI_PKG (these must always run because we need version info)
- Store parsed version in checkpoint marker

**Step 4: Version detection and checkpoint invalidation**
- Read installed version (existing logic)
- Compare against new version from APT index
- If version changed: call checkpoint_clear_all (except apt-index which we just set), print "Version change detected: {old} -> {new}, clearing checkpoints"
- If same version AND not forced AND no remaining work: skip with message

**Step 5: Resume detection**
- Call has_any_checkpoint; if true, set RESUMING=true, print banner: "=== Resuming from partial install (version {X}) ==="

**Step 6: Download LINBO .deb (checkpointed as "linbo-deb")**
- If checkpoint_exists "linbo-deb" AND checkpoint_version_match "linbo-deb" "${LINBO_VERSION}":
  - Print: "Skipping: LINBO .deb already downloaded ({version}, SHA256 OK)"
- Else:
  - Call download_and_cache_deb with LINBO_FILENAME, LINBO_SHA256, "linuxmuster-linbo7"
  - On failure: exit 1 (error_block already printed by download_and_cache_deb)
  - On success: checkpoint_set "linbo-deb" "${LINBO_VERSION}"

**Step 7: Download GUI .deb (checkpointed as "gui-deb")**
- Same pattern as Step 6 but for GUI package
- checkpoint_set "gui-deb" "${LINBO_VERSION}"

**Step 8: Extract and provision boot files (checkpointed as "boot-files")**
- If checkpoint_exists "boot-files": print skip message
- Else:
  - Extract LINBO .deb from cache: `dpkg-deb -x "${CACHE_DIR}/${LINBO_DEB_BASENAME}" "${TEMP_DIR}/linbo7"`
  - Call provision_boot_files with extract dir
  - On failure: error_block with details, exit 1
  - On success: checkpoint_set "boot-files" "${LINBO_VERSION}"
  - Extract GUI .deb from cache: `dpkg-deb -x "${CACHE_DIR}/${GUI_DEB_BASENAME}" "${TEMP_DIR}/gui7"`
  - Call provision_gui with extract dir
  - Write .needs-rebuild marker (existing logic)
  - Write version markers (existing logic)
  - Write .boot-files-installed marker (existing logic)
  - Set permissions (existing logic)
  - Cleanup temp dir

**Step 9: Provision kernels (checkpointed as "kernels")**
- If checkpoint_exists "kernels": print skip message
- Else:
  - Call provision_kernels
  - On success: checkpoint_set "kernels" "${LINBO_VERSION}"
  - Fix driver volume permissions (existing logic)

**Step 10: Provision themes (checkpointed as "themes")**
- If checkpoint_exists "themes": print skip message
- Else:
  - Call provision_themes
  - checkpoint_set "themes" "${LINBO_VERSION}"

**Step 11: Success summary**
- Calculate duration: `END_TIME=$(date +%s); DURATION=$((END_TIME - START_TIME))`
- Call print_success_summary "${LINBO_VERSION}" "${DURATION}"

**Error handling patterns throughout:**
- Use `if ! function_call; then exit 1; fi` instead of `function_call || exit 1` to ensure structured errors print before set -e kicks in
- For download failures: download_with_retry and download_and_cache_deb already call error_block internally
- For extraction failures: wrap dpkg-deb in `if ! dpkg-deb -x ...; then error_block "Extraction failed" ...; exit 1; fi`
- For provision failures: wrap provision_* calls similarly

**Also update the existing download_and_extract_deb function:**
- Keep it defined (existing code may reference it) but it is no longer called from the main flow
- The new flow uses download_and_cache_deb (download) + dpkg-deb -x (extract) separately

**Also update the existing fetch_packages_index:**
- Replace plain `echo "ERROR: ..."` with error_block call using classify_curl_error and run_network_diagnostics
- Keep function signature and return value unchanged

**Also update the existing verify_sha256:**
- Keep it as-is (backwards compat) -- new flow uses verify_sha256_structured instead

**Cache .deb file retention:**
- Downloaded .debs go to CACHE_DIR (e.g., /srv/linbo/.cache/linuxmuster-linbo7_4.3.31-0_amd64.deb)
- Do NOT rm the .deb after extraction (unlike current code which removes immediately)
- Cache is cleared only when FORCE_UPDATE=true (via checkpoint_clear_all which can also rm CACHE_DIR)
- Update checkpoint_clear_all to also remove CACHE_DIR contents

**TEMP_DIR usage:**
- Still use mktemp -d for extraction working directory (ephemeral, cleaned up in same run)
- Only create TEMP_DIR when extraction actually needed (not on resume with boot-files checkpoint)
