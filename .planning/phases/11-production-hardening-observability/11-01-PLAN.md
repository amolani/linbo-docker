<task type="auto">
  <name>Task 1: Create wait-ready.sh health gate and docker-compose resource limits</name>
  <files>scripts/wait-ready.sh, docker-compose.yml, Makefile</files>
  <action>
Create `scripts/wait-ready.sh` (~80 lines) that polls container health until all are ready or timeout:

1. **Shebang and setup:** `#!/bin/bash`, `set -euo pipefail`. Color-safe output using the same pattern as setup.sh (check `[[ -t 1 ]]` and `tput colors`).

2. **Configuration:** `TIMEOUT="${WAIT_TIMEOUT:-120}"`, `INTERVAL=3`. Define `HEALTH_SERVICES="cache api web tftp rsync ssh"` (NOT init -- it's one-shot).

3. **Init container check function:** `check_init()` uses `docker inspect --format '{{.State.ExitCode}}' linbo-init` to check exit code. If container is still running (State.Status=running), report "init still running". If exited with code 0, pass. If exited non-zero, report failure and exit 1. If container doesn't exist (first run before compose up), skip gracefully.

4. **Main polling loop:** `poll_health()` loops with `$INTERVAL` sleep until `$TIMEOUT`. Each iteration: for each service in `$HEALTH_SERVICES`, get status via `docker inspect --format '{{.State.Health.Status}}' "linbo-${svc}"`. If all return "healthy", print success message with elapsed time and exit 0. Handle DHCP container separately: check `docker inspect linbo-dhcp` exists first -- if it does, include it in health checks; if not (profile not activated), skip it.

5. **Timeout diagnostics:** If loop exits without all healthy, print "TIMEOUT after ${TIMEOUT}s" then for each unhealthy/missing container: print container name, current status, and last 5 log lines via `docker logs "linbo-${svc}" --tail 5 2>&1 | sed 's/^/    /'`. Exit 1.

6. **Entry point:** Call `check_init` first (early exit on init failure). Then call `poll_health`. Print final status.

Make the script executable: `chmod +x scripts/wait-ready.sh`.

Then modify `docker-compose.yml` to add `deploy.resources.limits` to ALL services (including init and dhcp). Use the values from research:
- init: cpus '2.0', memory 512M
- cache: cpus '1.0', memory 256M
- ssh: cpus '0.5', memory 128M
- rsync: cpus '2.0', memory 256M
- tftp: cpus '0.5', memory 64M
- api: cpus '2.0', memory 512M
- web: cpus '1.0', memory 128M
- dhcp: cpus '0.5', memory 64M

Place `deploy:` section at the end of each service definition (before depends_on if it has no other trailing sections, or after healthcheck). Follow the pattern:
```yaml
    deploy:
      resources:
        limits:
          cpus: '2.0'
          memory: 512M
```

Finally, add `wait-ready` target to Makefile in the "Health & Status" section:
```makefile
wait-ready:
	./scripts/wait-ready.sh
```
Add `wait-ready` to the `.PHONY` line. Add help text: `"  make wait-ready  - Wait until all containers are healthy"`.

<task type="auto">
  <name>Task 2: Create doctor.sh comprehensive diagnostics</name>
  <files>scripts/doctor.sh, Makefile</files>
  <action>
Create `scripts/doctor.sh` (~200 lines) that checks 6 diagnostic categories with PASS/FAIL output and fix suggestions:

1. **Shebang and setup:** `#!/bin/bash`, `set -euo pipefail`. Color-safe output (same pattern as wait-ready.sh). Initialize `PASS=0` and `FAIL=0` counters.

2. **check() helper function:** Takes 3 args: description, result (0=pass, non-zero=fail), fix (suggestion string). Prints `[PASS] description` in green or `[FAIL] description` in red. On fail, prints `       Fix: $fix` indented. Increments PASS/FAIL counters.

3. **Category 1 -- Container Health:** For each service in "cache api web tftp rsync ssh": run `docker inspect --format '{{.State.Health.Status}}' "linbo-${svc}" 2>/dev/null | grep -q healthy`. Use `check` with fix `"docker compose restart ${svc}"`. Handle DHCP like wait-ready: only check if container exists. Check init separately via exit code with fix `"docker compose up init"`.

4. **Category 2 -- Volume Permissions:** Test writable by API container: `docker exec linbo-api sh -c 'touch /srv/linbo/.doctor-test && rm /srv/linbo/.doctor-test' 2>/dev/null`. Fix: `"chown -R 1001:1001 $(docker volume inspect linbo_srv_data -f '{{.Mountpoint}}')"`. Only run if linbo-api container is running (check first with `docker inspect --format '{{.State.Running}}' linbo-api` -- skip with a warning if not running, do NOT count as FAIL for volume permissions).

5. **Category 3 -- SSH Key Presence:** Check 4 key files inside the SSH container via `docker exec linbo-ssh test -f <path>`:
   - `/etc/linuxmuster/linbo/ssh_host_rsa_key`
   - `/etc/linuxmuster/linbo/ssh_host_rsa_key.pub`
   - `/etc/linuxmuster/linbo/linbo_client_key`
   - `/etc/linuxmuster/linbo/linbo_client_key.pub`
   Fix for each: `"docker compose restart ssh (auto-generates keys on start)"`.
   Only run if linbo-ssh container is running.

6. **Category 4 -- linbofs64 Build Status:** Check inside API container:
   - `docker exec linbo-api test -f /srv/linbo/.linbofs-patch-status` with fix `"Trigger rebuild: curl -X POST http://localhost:3000/api/system/linbo-update/rebuild"`
   - `docker exec linbo-api test -f /srv/linbo/linbofs64` with fix `"Run: make rebuild-all"`
   Only run if linbo-api is running.

7. **Category 5 -- Redis Connectivity:** `docker exec linbo-cache redis-cli ping 2>/dev/null | grep -q PONG`. Fix: `"docker compose restart cache"`.

8. **Category 6 -- PXE Port Reachability:** Check ports on the HOST (not in containers):
   - TFTP 69/udp: `ss -ulnp sport = :69 2>/dev/null | grep -q 69`. Fix: `"TFTP uses host network. Check: docker compose restart tftp"`.
   - Rsync 873/tcp: `ss -tlnp sport = :873 2>/dev/null | grep -q 873`. Fix: `"docker compose restart rsync"`.
   - API 3000/tcp: `ss -tlnp sport = :3000 2>/dev/null | grep -q 3000`. Fix: `"docker compose restart api"`.
   - SSH 2222/tcp: `ss -tlnp sport = :2222 2>/dev/null | grep -q 2222`. Fix: `"docker compose restart ssh"`.

9. **Summary:** Print separator, then `"Results: N passed, M failed"`. If FAIL > 0, exit 1. Else exit 0.

10. **Section headers:** Print category names as section headers (e.g., `echo -e "\n${BLUE}Container Health${NC}"`) before each category's checks for readability.

Make executable: `chmod +x scripts/doctor.sh`.

Add `doctor` target to Makefile in the "Health & Status" section:
```makefile
doctor:
	./scripts/doctor.sh
```
Add `doctor` to the `.PHONY` line. Add help text: `"  make doctor      - Run system diagnostics"`.
