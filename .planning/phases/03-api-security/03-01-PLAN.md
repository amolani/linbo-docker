<objective>
Add JWT token and INTERNAL_API_KEY verification to the main WebSocket `/ws` endpoint upgrade handler, so unauthenticated connections are rejected with HTTP 401 before the WebSocket handshake completes.

Purpose: Currently, the `/ws` endpoint accepts any connection without authentication. This is a security gap -- any client can connect and receive real-time events (host status changes, operation updates, etc.) without proving identity.

Output: Modified upgrade handler in index.js with auth verification, plus unit tests covering all auth scenarios.
</objective>
