**3. Replace auth flow (the core change):**
Remove the admin/Muster! login entirely. Replace with:
```bash
# Read INTERNAL_API_KEY from remote .env
INTERNAL_KEY=$(ssh "$target" "grep '^INTERNAL_API_KEY=' ${REMOTE_DIR}/.env 2>/dev/null | sed 's/^INTERNAL_API_KEY=//'")

if [ -z "$INTERNAL_KEY" ]; then
  echo "  WARNING: INTERNAL_API_KEY not found in remote .env"
  echo "  Falling back to direct docker exec for rebuild..."
  ssh "$target" "docker exec linbo-api /usr/share/linuxmuster/linbo/update-linbofs.sh"
else
  # Use INTERNAL_API_KEY as Bearer token (authenticateToken accepts it)
  ssh "$target" "curl -sf -X POST http://localhost:3000/api/v1/system/update-linbofs \
    -H 'Authorization: Bearer $INTERNAL_KEY' \
    -H 'Content-Type: application/json'"
  # ... same for regenerate-grub-configs ...
fi
```

Use `sed 's/^INTERNAL_API_KEY=//'` instead of `cut -d= -f2` to handle values containing `=` (base64 keys).

IMPORTANT: Use `Authorization: Bearer $INTERNAL_KEY` (NOT `X-Internal-Key`) because system routes use `authenticateToken` middleware, not `authenticateInternal`. Per research finding.

**4. Add rsync container restart in --rebuild flow:**
After the GRUB regeneration step, restart rsync alongside tftp:
```bash
echo "=== Restarting TFTP + rsync ==="

```
