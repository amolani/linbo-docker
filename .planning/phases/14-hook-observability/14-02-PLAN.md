Implement two exported functions:

**readManifest():**
- Read `path.join(LINBO_DIR, '.linbofs-build-manifest.json')`
- Parse JSON, return parsed object
- On any error (file missing, invalid JSON): return null
- Simple try/catch, no logging needed

**getHooks():**
- Scan both `update-linbofs.pre.d` and `update-linbofs.post.d` directories
- For each file in each directory:
  - `fs.stat()` to get size, mode, mtime
  - `executable: !!(stat.mode & 0o111)` for executable check
  - Push `{ name, type ('pre'|'post'), path (full), executable, size }`
- Try/catch around readdir (directory might not exist)
- Sort hooks by name within each type
- Call `readManifest()` to get last build data
- For each hook, merge manifest entry (match by name + type):
  - Add `lastExitCode` and `lastFilesDelta` from manifest if found
- Return:
  ```javascript
  {
    hooks: [...],
    lastBuild: manifest?.buildTimestamp || null,
    hookWarnings: manifest?.hookWarnings || 0,
  }
  ```

**1. Create containers/api/src/routes/system/hooks.js:**

Follow the exact pattern from `linbofs.js`:
```javascript
const express = require('express');
const router = express.Router();
const { authenticateToken } = require('../../middleware/auth');
const hookService = require('../../services/hook.service');

router.get('/hooks', authenticateToken, async (req, res, next) => {
  try {
    const hooks = await hookService.getHooks();
    res.json({ data: hooks });
  } catch (error) {
    next(error);
  }
});

module.exports = router;
```

No `requireRole` needed — read-only endpoint, any authenticated user can view hook status (consistent with GET /system/linbofs-status which also uses only authenticateToken).

**3. Add build log rotation to containers/api/src/services/linbofs.service.js:**

Add a `rotateBuildLogs()` function:
- Scan LINBO_DIR for files matching `.linbofs-build` prefix and `.log` extension
- Use `fs.readdir()` + `fs.stat()` to get file list with mtime
- Sort by mtime descending (newest first)
- Delete all logs beyond the 3 most recent using `fs.unlink()`
- Silently ignore errors (log file might be deleted by another process)

Modify `updateLinbofs()`:
- BEFORE the exec call: call `await rotateBuildLogs()`
- AFTER the exec call (whether success or failure): write the build output to a timestamped log file:
  - Filename: `.linbofs-build.${new Date().toISOString().replace(/[:.]/g, '-')}.log`
  - Content: the stdout output from the exec call
  - Use `fs.writeFile()` with `catch(() => {})` (non-critical, don't fail the build)

Add `rotateBuildLogs` to exports (useful for testing, though not required by API).

**IMPORTANT:** The `updateLinbofs()` function currently uses `execAsync`. The log capture fits naturally — `stdout` is already captured in the return value. Just write it to the log file after execution completes.
