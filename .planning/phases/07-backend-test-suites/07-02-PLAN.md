**Mock setup (module-level, before requires):**

1. **fs** -- `jest.mock('fs', () => ({ readFileSync: jest.fn(() => Buffer.from('mock-ssh-key')) }))` -- MUST be before require of terminal.service because module loads key at require-time (lines 17-31).

2. **ssh2** -- `jest.mock('ssh2', ...)` using EventEmitter-based mock. Use `setTimeout(() => this.emit('ready'), 0)` (not 10ms) for deterministic timing. The mock must support reconfiguration per test:
   - Default: shell() succeeds (PTY mode)
   - PTY-fail variant: shell(opts, cb) calls cb(new Error('PTY not available'))
   - Connection-fail variant: connect() emits 'error' instead of 'ready'

   Store the most recently created mock client in a module-scoped variable (`let lastMockClient`) so tests can access it for assertions (e.g., verifying client.end was called).

3. **uuid** -- `jest.mock('uuid', () => ({ v4: jest.fn(() => 'mock-session-id-' + (++uuidCounter)) }))` with a counter reset in beforeEach for predictable IDs.

**Timer setup:** Use `jest.useFakeTimers()` for idle timeout tests, `jest.useRealTimers()` in afterEach/afterAll.

**Key implementation notes per CONTEXT.md decisions:**
- Use jest.useFakeTimers() + jest.advanceTimersByTime() for idle timeout -- no real timer waits
- ssh2.Client mock uses EventEmitter from 'events' module
- Mock fs.readFileSync BEFORE requiring terminal.service (module-level key load at lines 17-31)
- Call destroyAll() in beforeEach to clean up sessions Map between tests
- The ssh2 mock's connect() must use `setTimeout(() => this.emit('ready'), 0)` for deterministic async behavior. Tests must `await` the createSession promise.
- Set `process.env.TERMINAL_MAX_SESSIONS = '2'` at top of file for manageable max-sessions test

**Async considerations:**
- createSession returns a Promise. The ssh2 mock emits 'ready' via setTimeout(fn, 0). After calling createSession, use `await` which naturally waits for the promise resolution.
- For idle timeout tests with fake timers: create session (which involves setTimeout for the ready event), then advance timers. Since jest.useFakeTimers() controls setTimeout, the 'ready' emission may need manual flushing. Use `jest.advanceTimersByTime(1)` to trigger the ready event, then `await` the createSession promise, then advance by IDLE_TIMEOUT_MS.
