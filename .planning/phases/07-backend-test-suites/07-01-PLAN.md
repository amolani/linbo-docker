**Test structure (describe/it blocks):**

```
describe('getRemoteManifest()')
  - it('fetches manifest from API and caches in Redis')
  - it('returns cached manifest from Redis when available')

describe('getLocalImages()')
  - it('scans directories and returns image list')
  - it('returns empty array when IMAGES_DIR does not exist')

describe('pullImage()')
  - it('acquires lock and starts download when no other job running')
  - it('queues job when lock already held')

describe('pullImage() -- resume download')
  - it('sends Range header when .part file exists')
    Mock: fsp.stat resolves with { size: 500 } for .part file.
    Mock: fetch HEAD returns content-length 1000, etag '"abc"'.
    Mock: fetch GET returns 206.
    Verify: fetch called with headers.Range = 'bytes=500-' and headers['If-Range'] = '"abc"'.
    Use `await new Promise(r => setTimeout(r, 0))` to flush microtasks after pullImage.

describe('pullImage() -- MD5 verification')
  - it('completes when MD5 matches')
    Mock: fs.existsSync returns true for .md5 path.
    Mock: fsp.readFile returns 'abcdef1234567890abcdef1234567890  image.qcow2'.
    Mock: crypto.createHash -> createReadStream flow returns matching hash.
    For _computeMd5 mocking: mock fs.createReadStream to return an EventEmitter that emits 'data' then 'end'. The real crypto.createHash('md5') can be used since it's CPU-only.

  - it('fails when MD5 mismatches')
    Same setup but fsp.readFile returns different hash.
    After flush, verify job status set to 'failed' with 'MD5 mismatch' error.

describe('pullImage() -- atomic swap')
  - it('removes target dir then renames staging dir')
    Mock: fs.existsSync returns true for targetDir.
    After download completes, verify: fsp.rm called with targetDir + {recursive: true, force: true}, then fsp.rename called with stagingDir -> targetDir.

describe('pullImage() -- queue ordering')
  - it('second pull is queued and starts after first completes')
    First pullImage acquires lock (NX succeeds). Second pullImage gets null from NX, rpush to queue.
    After first download completes (lock released), _processNextInQueue does lpop and starts next.
    Verify: rpush called for second job, lpop called after first finishes.

describe('cancelJob()')
  - it('cancels running job by aborting download')
  - it('removes queued job from queue')
  - it('returns not found for unknown jobId')

describe('recoverOnStartup()')
  - it('cleans stale lock and marks job as failed')
  - it('starts next queued job after recovery')

describe('edge cases')
  - it('handles network failure mid-download')
    Mock pipeline to reject with Error('ECONNRESET'). Verify job status set to 'failed'.
  ```

- Mock `stream/promises` pipeline to resolve immediately -- don't test stream internals
