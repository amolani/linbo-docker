This is a mechanical extraction. Follow these steps exactly:

**Step 0: Delete system.js FIRST.**
Node.js resolves `require('./system')` to `system.js` before `system/index.js`. The old file MUST be gone before the directory exists. Delete `containers/api/src/routes/system.js`, then create the `containers/api/src/routes/system/` directory.

**Step 1: Create aggregator `routes/system/index.js`.**
```javascript
const express = require('express');
const router = express.Router();

router.use('/', require('./linbofs'));
router.use('/', require('./kernel'));
router.use('/', require('./firmware'));
router.use('/', require('./wlan'));
router.use('/', require('./grub-theme'));
router.use('/', require('./grub-config'));
router.use('/', require('./worker'));
router.use('/', require('./linbo-update'));

module.exports = router;
```
Mount order matches the original section order in system.js. All sub-routers mount at `'/'` because the route paths are already fully qualified (e.g., `/update-linbofs`, `/kernel-variants`). The `/system` prefix comes from `routes/index.js`.

**Step 2: Create each sub-router file.**

Each file follows the same boilerplate pattern:
```javascript
const express = require('express');
const router = express.Router();
const { authenticateToken, requireRole } = require('../../middleware/auth');
const { auditAction } = require('../../middleware/audit');
// ... service-specific imports (see dependency map in interfaces block above)

// ... Zod schemas (if any) ...
// ... route handlers ...

module.exports = router;
```

**CRITICAL: Import paths change.** All imports that were `../something` in system.js become `../../something` in sub-routers (one directory deeper).

- Do NOT change route registration order within the aggregator. Express evaluates routes in registration order.
