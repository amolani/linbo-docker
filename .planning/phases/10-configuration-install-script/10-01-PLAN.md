  <action>
Create `setup.sh` at the project root (NOT in scripts/) as a bash script with `#!/bin/bash` and `set -euo pipefail`.

**Structure the script in this order:**

1. **Header and color setup** (~20 lines)
   - Colors: RED, GREEN, YELLOW, BLUE, NC (same as scripts/install.sh)
   - `SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"` for reliable path resolution
   - Color-safe output: check `tput colors` or TERM to gracefully degrade in non-interactive terminals (pipes, CI). If no color support, set all color vars to empty string.

2. **Helper functions** (~30 lines)
   - `print_check()` taking status ("PASS"/"FAIL"/"WARN"), name, detail. Format: `[PASS] Docker ................. version 24.0.7` with dots padding to align output. Use GREEN for PASS, RED for FAIL, YELLOW for WARN.
   - `print_banner()` -- simplified LINBO Docker Setup banner (shorter than install.sh's ASCII art, just a clean header box)
   - `log_info()`, `log_error()`, `log_warn()` -- one-line helpers matching install.sh style

3. **Prerequisite checks** (~80 lines) -- BOOT-02
   All checks run to completion (don't exit on first failure). Track `PREREQ_FAILED=0`, increment on failure. Exit with summary if any failed.

   - `check_root()` -- must be root or in docker group. If not root, warn that `ss -p` process names may be unavailable. If not root AND not in docker group, FAIL.
   - `check_docker()` -- `command -v docker` exists, `docker version --format '{{.Server.Version}}'` succeeds (daemon running). FAIL with fix: `curl -fsSL https://get.docker.com | sh`
   - `check_compose()` -- `docker compose version` succeeds. FAIL with fix: `apt-get install docker-compose-plugin`
   - `check_disk()` -- `df -P /srv 2>/dev/null || df -P /` available space >= 2GB (2097152 KB). FAIL with fix: "Free up disk space in /srv"
   - `check_dns()` -- `nslookup deb.linuxmuster.net` or `host deb.linuxmuster.net` or `getent hosts deb.linuxmuster.net`. Try each in order, PASS on first success. FAIL with fix: "Check /etc/resolv.conf or set DNS server"
   - `check_network()` -- `curl -sf --max-time 10 https://deb.linuxmuster.net/dists/lmn73/Release -o /dev/null`. FAIL with fix: "Check internet connectivity or firewall/proxy settings"
   - `check_openssl()` -- `command -v openssl` exists. FAIL with fix: `apt-get install openssl`. Fallback note: if openssl missing, secrets can use `/dev/urandom` but warn.

4. **Port conflict detection** (~40 lines) -- ERR-03
   Run after prerequisites pass.

   - `check_port_conflict()` taking port, proto (tcp/udp), service name
   - Use `ss -ulnp sport = :69` for TFTP, `ss -tlnp sport = :873` for rsync
   - Parse process name: `grep -oP 'users:\(\("\K[^"]+'` from ss output
   - For TFTP (port 69): "TFTP uses host networking and CANNOT be remapped. Stop the conflicting service: systemctl stop <process> && systemctl disable <process>"
   - For rsync (port 873): "Stop the conflicting service: systemctl stop <process>" (can theoretically remap but typically not done)
   - If process is "docker-proxy": "Another Docker container is using this port. Run: docker ps --filter publish=<port>"
   - Port checks are WARNING, not FAIL -- admin may want to configure now and stop the service later before `docker compose up`. Set `PORT_WARNINGS=1` and display summary.

5. **IP auto-detection** (~30 lines) -- BOOT-03
   - `detect_server_ip()` -- Method 1: `ip route get 1.1.1.1 2>/dev/null | awk '{for(i=1;i<=NF;i++) if($i=="src") print $(i+1)}'` (safer than `$7` which can shift). Method 2: `hostname -I | awk '{print $1}'`. Method 3: fallback "127.0.0.1"
   - Show detected IP, list ALL available interface IPs with interface names using `ip -4 addr show | grep -oP '(?<=inet\s)\d+(\.\d+){3}' + interface name` so admin can pick the right one on multi-homed servers
   - `read -p "Use this IP for LINBO_SERVER_IP? [Y/n]: "` -- Enter or Y accepts, N prompts for manual input
   - Validate entered IP with regex: `^[0-9]+\.[0-9]+\.[0-9]+\.[0-9]+$`

6. **Secret generation** (~20 lines) -- BOOT-04
   - `generate_secret()` taking type: jwt -> `openssl rand -base64 48 | tr -d '\n'`, api_key -> `openssl rand -hex 32`, password -> `openssl rand -base64 24 | tr -d '\n/+=' | head -c 32`
   - Fallback if openssl not available (should have been caught by prereq, but defense in depth): `head -c 48 /dev/urandom | base64 | tr -d '\n'`
   - Generate: JWT_SECRET, INTERNAL_API_KEY, DB_PASSWORD, RSYNC_PASSWORD

7. **GITHUB_TOKEN prompt** (~15 lines)
   - Explain: "The web UI container requires a GitHub token for npm package access."
   - Explain: "If you don't have one, the web container build will fail (tracked as OSS-01)."
   - `read -p "GitHub token (leave empty to skip): "` -- accept empty
   - If empty, set GITHUB_TOKEN="" and warn

8. **Existing .env detection** (~15 lines)
   - If `.env` exists: show last modified date, offer backup + overwrite
   - Backup format: `.env.backup.YYYYMMDD-HHMMSS`
   - If admin declines overwrite, exit gracefully

9. **.env file write** (~40 lines) -- BOOT-01
   - Write to `.env.tmp` first, then `mv .env.tmp .env` (atomic)
   - `chmod 600 .env` (contains secrets)
   - Include header with generation timestamp
   - Variable sections with comments:
     - `# === Server ===` : LINBO_SERVER_IP, NODE_ENV=production
     - `# === Secrets (auto-generated -- do not share) ===` : JWT_SECRET, INTERNAL_API_KEY, DB_PASSWORD, RSYNC_PASSWORD
     - `# === Web UI ===` : ADMIN_USERNAME=admin, ADMIN_PASSWORD=Muster!, GITHUB_TOKEN
     - `# === Network (uncomment to override defaults) ===` : commented-out LINBO_SUBNET, LINBO_NETMASK, LINBO_GATEWAY, LINBO_DNS, LINBO_DOMAIN
     - `# === Optional (uncomment to override defaults) ===` : commented-out WEB_PORT, API_PORT, LOG_LEVEL
   - Use quoted heredoc (`<< 'ENVEOF'`) for the template structure, with explicit variable substitution for dynamic values. This prevents accidental shell expansion.

10. **rsyncd.secrets sync** (~10 lines)
    - Write `config/rsyncd.secrets` with format `linbo:<RSYNC_PASSWORD>`
    - `chmod 600 config/rsyncd.secrets`
    - Print note about matching password

11. **Validation and summary** (~20 lines)
    - Run `docker compose config --quiet` to validate the generated .env (suppresses output, exits non-zero on error)
    - Print success summary: .env location, key values (IP, admin user -- NOT secrets), next steps (`make up` or `docker compose up -d`)
    - If PORT_WARNINGS was set, remind about port conflicts to resolve before starting

12. **main() flow** -- orchestrates all the above in order:
    `print_banner -> check_root -> run_prerequisites -> check_ports -> detect_ip_interactive -> generate_secrets -> prompt_github_token -> handle_existing_env -> write_env -> write_rsyncd_secrets -> validate_and_summarize`

Make the script executable: the Write tool creates the file, executor should `chmod +x setup.sh`.

**Important implementation notes:**
- Do NOT use `set -e` inside functions that are expected to return non-zero (prereq checks). Use explicit return values and track failures with a counter.
- The script runs on the HOST (full Linux, not Alpine/busybox). Bash 5.x features like arrays and `read -p` are available.
- All `read` prompts must handle non-interactive mode gracefully (detect with `[[ -t 0 ]]`). In non-interactive mode, use defaults and auto-detected values without prompting.
- The script does NOT start containers. It only configures. The admin runs `make up` or `docker compose up -d` separately.
