**1. Pin LMN original (DIFF-01):**
Extract the LMN original update-linbofs from `docs/linuxmuster-cachingserver-linbo7-main.zip` at path `linuxmuster-cachingserver-linbo7-main/serverfs/usr/sbin/update-linbofs`. Save as `scripts/server/update-linbofs-lmn-original.sh`. Prepend a reference header block:

```
#!/bin/bash
#
# LMN ORIGINAL - DO NOT EXECUTE IN DOCKER
# ========================================
# This is the original update-linbofs script from linuxmuster-cachingserver-linbo7
# package version 4.3.31-0 (2026-02-10), pinned as reference for diff comparison.
#
# Source: https://github.com/linuxmuster/linuxmuster-cachingserver-linbo7
# Path:   serverfs/usr/sbin/update-linbofs
#
# This script requires the full linuxmuster.net stack (helperfunctions.sh,
# constants.py, setup.ini) and WILL NOT RUN in a Docker environment.
#
# Purpose: Compare with scripts/server/update-linbofs.sh to understand what
# Docker's build pipeline does differently.
#
# LMN variable mappings (from constants.py):
#   LINBODIR      = /srv/linbo
#   LINBOSYSDIR   = /etc/linuxmuster/linbo
#   LINBOVARDIR   = /var/lib/linuxmuster/linbo
#   LINBOCACHEDIR = /var/cache/linuxmuster/linbo
#   LINBOSHAREDIR = /usr/share/linuxmuster/linbo
#   HOOKSDIR      = /var/lib/linuxmuster/hooks
#   LINBOLOGDIR   = /var/log/linuxmuster/linbo
#
# === ORIGINAL SCRIPT FOLLOWS ===
```

Then append the original script content. Make the file NOT executable (chmod 644) -- it is reference only.

**2. Create linbofs-audit.sh (DIFF-02):**
Create `scripts/server/linbofs-audit.sh` following the `doctor.sh` style (set -euo pipefail, section headers with `===`). The script inspects the built linbofs64 at `/srv/linbo/linbofs64` and outputs:

Sections:
- **Header:** "LINBO Docker - linbofs64 Audit" with date and file path
- **Archive Info:** File size (human-readable), MD5 hash, last modified timestamp
- **Kernel:** Version extracted from module paths (`lib/modules/X.Y.Z`), total module count (.ko + .ko.xz)
- **SSH Keys:** Fingerprints using `ssh-keygen -lf` for files matching `etc/ssh/ssh_host_*_key` and `etc/dropbear/dropbear_*_host_key`. For dropbear keys, try `dropbearkey -y -f` to get the public key, then `ssh-keygen -lf -`. If that fails, show filename + size.
- **Firmware:** Count and list of firmware files under `lib/firmware/`
- **Hook-Modified Files:** If template linbofs64.xz exists at `/var/lib/linuxmuster/linbo/current/linbofs64.xz`, compare file lists (template vs built) and show files only in built (labeled as "ADDED by Docker pipeline + hooks"). If no template, note "Template not available -- cannot determine hook modifications."
- **Device Nodes:** List entries from `cpio -t` that match `dev/` prefix
- **Summary:** Total file count, total module count, firmware count

Implementation notes:
- Extract linbofs64 using `xzcat "$LINBOFS" | cpio -t 2>/dev/null` for listing, `cpio -i -d -H newc --no-absolute-filenames` for extraction
- Check file existence before inspecting; print clear error if linbofs64 not built yet
- Use a temp directory with `trap "rm -rf ..." EXIT` for SSH key extraction
- Exit 0 on success, exit 1 if linbofs64 not found

**3. Create linbofs-diff.sh (DIFF-03):**
Create `scripts/server/linbofs-diff.sh` that compares template linbofs64.xz vs built linbofs64.

Output sections:
- **Header:** "LINBO Docker - linbofs64 Diff" with date
- **File Sizes:** Template size vs built size (human-readable)
- **ADDED** (files only in built): categorized by type -- modules (.ko/.ko.xz), firmware (lib/firmware/), SSH keys (etc/ssh/, etc/dropbear/), themes (usr/share/linbo/gui-themes/), other
- **REMOVED** (files only in template): list with count
- **SUMMARY:** Total added, removed, common file counts

Implementation notes:
- Use `cpio -t` to generate file lists from both archives (NOT extraction -- faster and works for device nodes)
- Use `comm -13`, `comm -23`, `comm -12` on sorted file lists for added/removed/common
- Check both files exist. If template missing, print: "Template not found at /var/lib/linuxmuster/linbo/current/linbofs64.xz -- run init container first."
- If built missing, print: "Built linbofs64 not found at /srv/linbo/linbofs64 -- run update-linbofs.sh first."
- Use temp directory with trap cleanup
- Exit 0 on success, exit 1 if either file missing

Both scripts: `chmod +x` and use `#!/bin/bash` shebang.
