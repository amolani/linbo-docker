**Section 1: Error Reporting Functions**

1. `error_block()` -- Takes 5 args: title, details, cause, diagnostics (optional, empty string if none), fix. Prints structured error block to stderr (>&2). Format MUST match CONTEXT.md:
   ```
   === ERROR: {title} ===
   {details}
   Cause:   {cause}

   Diagnostics:
     {diagnostics}

   Fix: {fix}

   To retry:  docker compose up init
   To reset:  FORCE_UPDATE=true docker compose up init
   ===========================================
   ```
   Use multiple `echo` calls (not printf with \n) for POSIX safety. Skip Diagnostics section if diagnostics arg is empty.

2. `classify_curl_error()` -- Takes 1 arg (curl exit code). Echoes human-readable cause string. Map at minimum: 5=proxy resolve, 6=DNS, 7=connection refused, 22=HTTP error, 28=timeout, 35=TLS, 47=redirects, 52=empty response, 56=transfer failed, *=generic with exit code.

3. `run_network_diagnostics()` -- Takes 1 arg (hostname). Runs nslookup check, detects HTTP_PROXY/HTTPS_PROXY. Echoes multi-line diagnostics string (each line prefixed with "  "). Uses multiple echo calls for portability.

**Section 2: Pre-flight Check Functions**

4. `check_disk_space()` -- Uses `df -P` to check available space on LINBO_DIR. Fails with error_block if <500MB. On success prints "  Disk space: {N}MB available (>= 500MB required)".

5. `check_dns()` -- Runs `nslookup deb.linuxmuster.net`. On failure, calls error_block with DNS error and network diagnostics. On success prints "  DNS: deb.linuxmuster.net resolves OK".

6. `check_write_permission()` -- Takes 1 arg (path). Attempts `touch {path}/.write-test`. On failure, uses `stat -c '%u:%g'` to get current ownership, calls error_block with EACCES error showing current vs expected (1001:1001) ownership and chown fix command. Cleans up .write-test on success.

**Section 3: Checkpoint Functions**

7. `checkpoint_exists()` -- Takes 1 arg (step name). Returns 0 if marker file exists in CHECKPOINT_DIR.

8. `checkpoint_set()` -- Takes 2 args (step name, version). Creates marker file with key=value format: version={version}, timestamp={ISO 8601 UTC}. Uses atomic write (temp file + mv). Creates CHECKPOINT_DIR if needed.

9. `checkpoint_clear_all()` -- Removes entire CHECKPOINT_DIR.

10. `checkpoint_version_match()` -- Takes 2 args (step name, expected version). Returns 0 if checkpoint exists AND its version matches expected. Used for version-based invalidation.

11. `has_any_checkpoint()` -- Returns 0 if any checkpoint marker files exist. Used for resume banner detection.

**Section 4: Enhanced Download and Verify Functions**

12. `verify_sha256_structured()` -- Takes 3 args (file, expected hash, label). On mismatch, calls error_block with SHA256 error showing expected vs actual. On success prints "  SHA256 OK: {basename}". Replaces the old verify_sha256 (keep old one for backwards compat but it won't be called from the new flow).

13. `download_with_retry()` -- Takes 2 args (url, output path). 3 retries with 5s sleep between. On exhaustion, captures curl exit code, calls classify_curl_error, runs run_network_diagnostics on the hostname, calls error_block. Returns 1. On success returns 0.

14. `download_and_cache_deb()` -- Takes 4 args (filename, sha256, label, cache_subdir). Downloads to CACHE_DIR (persistent volume, NOT /tmp). If cached file exists with matching SHA256, skips download and prints skip message. After download, verifies SHA256 with verify_sha256_structured. Sets checkpoint after successful download+verify. Does NOT extract (extraction happens separately). Creates CACHE_DIR if needed.

**Section 5: Success Summary**

15. `print_success_summary()` -- Takes 2 args (version, duration in seconds). Collects kernel versions from LINBO_DIR/kernels/{stable,longterm,legacy}/version, GUI status from linbo_gui64_7.tar.lz existence, themes from gui-themes subdirs. Prints summary block matching CONTEXT.md format:
    ```
    === LINBO Init Complete ===
    Version:  {version}
    Kernels:  stable ({ver}), longterm ({ver}), legacy ({ver})
    GUI:      linbo_gui64_7.tar.lz installed
    Themes:   edulution
    Duration: {N}s
    ===========================
    ```

All functions use local-style variable naming with underscore prefix (e.g., `_title`, `_path`) consistent with existing code patterns. All error output goes to stderr via >&2. All functions are POSIX sh compatible (no bashisms).

Do NOT modify any existing functions or the main flow in this task. The new functions are purely additive.
