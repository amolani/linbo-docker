Create `docs/INSTALL.md` in German prose with English technical terms. This is a LINEAR, procedural document — each section builds on the previous.

**Structure (follow this section order exactly):**

**1. Einleitung** — 2-3 sentences: what LINBO Docker is, what this guide covers (sync mode with LMN server), link to ADMIN-GUIDE.md for architecture details.

**2. Voraussetzungen**
- Hardware: 4 GB RAM minimum, 50 GB Festplatte (mehr fuer Images), Netzwerk-Interface im PXE-Subnet
- Software: Ubuntu 22.04/24.04 oder Debian 12
- Docker Engine install (if not present): link to official Docker docs `https://docs.docker.com/engine/install/` + quick snippet for Ubuntu:
  ```bash
  curl -fsSL https://get.docker.com | sh
  sudo usermod -aG docker $USER
  ```
- Docker Compose v2 plugin (included in modern Docker Engine)
- Netzwerkzugang zu `deb.linuxmuster.net` (HTTPS, Port 443)
- openssl (fuer Secret-Generierung)

**3. Installation**
- `git clone https://github.com/amolani/linbo-docker.git && cd linbo-docker`
- `./setup.sh` ausfuehren — explain what it does:
  - 7 Prerequisite-Checks (root/docker, Docker, Compose, Disk >= 2GB, DNS, Network, openssl)
  - Port-Konflikt-Pruefung (69/udp, 873/tcp)
  - Automatische IP-Erkennung auf dem PXE-Interface
  - Sichere Secret-Generierung (JWT_SECRET, INTERNAL_API_KEY, DB_PASSWORD, RSYNC_PASSWORD)
  - GITHUB_TOKEN-Abfrage (noetig fuer Web-Container npm packages, see OSS-01 for future removal)
  - Schreibt `.env` (Mode 600) und `config/rsyncd.secrets`
- `.env` Ueberblick: Tabelle der wichtigsten Variablen aus .env.example mit Erklaerung (LINBO_SERVER_IP, JWT_SECRET, GITHUB_TOKEN, ADMIN_USERNAME/PASSWORD, SYNC_ENABLED, LMN_API_URL, LMN_API_KEY, WEB_PORT, API_PORT)
- Note: `ADMIN_PASSWORD=Muster!` ist das Standard-Passwort — im Web-UI unter Settings aendern

**4. Erster Start**
- `docker compose up -d`
- `make wait-ready` — erklaere was passiert:
  - Init-Container: laedt linbo7 .deb von deb.linuxmuster.net, extrahiert Boot-Dateien, Kernel, GRUB
  - SSH-Container: generiert automatisch SSH/Dropbear-Keys
  - Cache-Container: Redis startet
  - API-Container: baut linbofs64 (injiziert SSH-Keys + Kernel-Module), erzeugt GRUB-Configs
  - TFTP-Container: wartet auf `.linbofs-patch-status` Marker, dann ready
  - Web-Container: startet Nginx mit React-SPA
- Erster Start dauert 2-5 Minuten (abhaengig von Internet-Geschwindigkeit)

**5. Verifikation**
- `make doctor` — erklaere die 6 Kategorien kurz (Container Health, Volume Permissions, SSH Keys, linbofs64 Build Status, Redis Connectivity, PXE Port Reachability)
- Web-UI: `http://<LINBO_SERVER_IP>:8080` — Login `admin` / `Muster!`
- API Health: `curl -sf http://localhost:3000/health` — muss `{"status":"ok"}` zurueckgeben
- Alle 24 Checks muessen PASS zeigen

**6. DHCP-Konfiguration**
- Erklaere: LINBO Docker bringt KEINEN eigenen DHCP mit (ausser optional). PXE-Boot braucht DHCP-Optionen.
- **Option A: Bestehender DHCP-Server** — ISC DHCP config snippet (aus RESEARCH.md verifiziert):
  ```
  option architecture-type code 93 = unsigned integer 16;
  if option architecture-type = 00:07 {
      filename "boot/grub/x86_64-efi/core.efi";
  } elsif option architecture-type = 00:09 {
      filename "boot/grub/x86_64-efi/core.efi";
  } else {
      filename "boot/grub/i386-pc/core.0";
  }
  next-server <LINBO_SERVER_IP>;
  ```
  - Auch dnsmasq-Snippet (fuer Schulen die dnsmasq nutzen):
  ```
  dhcp-match=set:bios,option:client-arch,0
  dhcp-match=set:efi64,option:client-arch,7
  dhcp-match=set:efi64,option:client-arch,9
  dhcp-boot=tag:bios,boot/grub/i386-pc/core.0,<LINBO_SERVER_IP>
  dhcp-boot=tag:efi64,boot/grub/x86_64-efi/core.efi,<LINBO_SERVER_IP>
  ```
- **Option B: Eingebauter Proxy-DHCP-Container** — `docker compose --profile dhcp up -d`
  - Erklaere: proxy-DHCP ergaenzt bestehenden DHCP, vergibt keine IPs, nur PXE-Optionen
  - Konfiguration: DHCP_INTERFACE in .env setzen
  - Vorteil: kein bestehender DHCP-Server muss modifiziert werden
- BIOS vs UEFI: kurze Erklaerung welche Boot-Datei fuer was (`core.0` = BIOS, `core.efi` = UEFI)

**7. Erster PXE-Boot (Verifikation)**
- Testclient auf PXE-Boot konfigurieren (BIOS/UEFI Boot-Reihenfolge)
- Erwartete Boot-Sequenz: DHCP -> GRUB (TFTP) -> Kernel + linbofs64 (HTTP) -> init.sh -> linbo_gui
- Was bei jedem Schritt zu sehen ist:
  - GRUB-Menue: LINBO Boot-Eintrag
  - Kernel-Meldungen (Netzwerk-Treiber laden)
  - LINBO-GUI erscheint mit Gruppen-Buttons
- Wenn es haengt: welchen Schritt pruefen (DHCP? TFTP? Kernel-Module?)

**8. Sync-Modus einrichten**
- Authority API auf dem LMN-Server (kurze Erklaerung was es ist)
- In `.env` setzen:
  ```
  SYNC_ENABLED=true
  LMN_API_URL=http://10.0.0.11:8001
  LMN_API_KEY=<api-key>
  ```
- `docker compose up -d` (neustart mit neuer Config)
- Verifikation: `curl -sf http://localhost:3000/api/v1/sync/status | jq` — muss `connected` zeigen
- Erklaere: Im Sync-Modus ist Docker read-only fuer LMN-Daten (Hosts, Configs, Rooms kommen vom LMN-Server)

**9. Haeufige Probleme**
- Inline die Top 5 Probleme:
  1. Port 69/udp belegt (anderer TFTP-Server) — `ss -ulnp | grep :69` + Loesung
  2. Permission-Fehler (EACCES) — `docker exec linbo-api ls -la /srv/linbo` + `chown -R 1001:1001`
  3. .env-Aenderungen nicht aktiv — `docker compose up -d` statt `docker compose restart`
  4. Web-Container Build scheitert 401 — GITHUB_TOKEN in .env pruefen
  5. TFTP liefert leeres linbofs64 — Warten auf `.linbofs-patch-status` Marker, `make doctor` pruefen
- Link: "Ausfuehrliche Fehlerdiagnose: [docs/TROUBLESHOOTING.md](TROUBLESHOOTING.md)"

**10. Naechste Schritte**
- Link to ADMIN-GUIDE.md fuer Architektur-Details und Wartung
- Link to hooks.md fuer Anpassungen (z.B. Plymouth Theme)
- Link to UNTERSCHIEDE-ZU-LINBO.md fuer Vergleich mit Vanilla-LINBO
